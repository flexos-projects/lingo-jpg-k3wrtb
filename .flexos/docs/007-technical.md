# Technical

## Stack
| Layer | Choice | Rationale |
|-------|--------|-----------|
| Framework | Nuxt 4 (Vue) | Fast, modern, builds a beautiful PWA that feels native without App Store delays. Excellent route transitions. |
| Styling | CSS Modules / Tokens | Native CSS with FlexOS token system for predictable theming. Avoids heavy CSS-in-JS runtime. |
| Database | Supabase (PostgreSQL) | Relational data for economy + real-time sync for points and comments. Built-in Auth. |
| Auth | Supabase Auth | Frictionless social login (Google/Apple) to prevent drop-off. |
| AI | Anthropic Claude 3.5 Sonnet | Superior at parsing cultural context, nuance, and humor compared to standard translation models. |
| Hosting | Vercel | Edge network delivery ensures images and API routes load instantly globally. |
| Storage | Vercel Blob | Handles massive volume of UGC meme images with on-the-fly optimization (WebP). |

## Architecture
- **Rendering Strategy:** SPA mode for the authenticated app (`/feed`, `/vault`, `/deck`) to ensure zero-latency swiping, persistent audio context, and uninterrupted haptics. SSR for the public landing page (`/`) for SEO and fast initial load.
- **Data Flow:** Browser → Nuxt Server API (`/api/*`) → Supabase / Anthropic. The browser never calls Supabase or Anthropic directly to protect secrets, enforce tier limits, and ensure atomic transactions for the points economy.
- **Feed Caching:** The client pre-fetches the next 5 memes in the background. Images are aggressively cached using a Service Worker to ensure the swipe-up animation never stutters waiting for network.

## Key Configuration
- Nuxt configured for PWA (manifest, service worker for caching images, install prompts).
- FlexOS conventions: explicit component registration, no auto-imports for critical state to ensure predictability.

## Environment Variables
- `NUXT_PUBLIC_SUPABASE_URL` (Public) - Supabase endpoint.
- `NUXT_PUBLIC_SUPABASE_ANON_KEY` (Public) - Supabase anon key.
- `SUPABASE_SERVICE_ROLE_KEY` (Server) - Admin overrides for economy transactions.
- `ANTHROPIC_API_KEY` (Server) - Claude API access.
- `BLOB_READ_WRITE_TOKEN` (Server) - Vercel Blob access.

## Auth & Security
- Supabase Auth with OAuth providers.
- Sessions stored in httpOnly cookies (managed by Nuxt server routes) for security against XSS.
- 30-day expiry.
- **Economy Security:** Points balance is mutated via a secure PostgreSQL function (`increment_points`), never via direct client updates.
- **RLS Policies:** Users can only read their own `attempts` and `saves`.

## API Design
- `GET /api/memes/feed` - Returns batch of 5 memes, cursor-based pagination. Excludes memes the user has already attempted.
- `POST /api/attempts` - Submits a guess, calculates points server-side, returns updated balance and breakdown.
- `POST /api/ugc/analyze` - Accepts image upload, returns Claude 3.5 analysis.
- `POST /api/vault/purchase` - Atomically deducts points and creates unlock record.

## Key Integrations & Fallbacks
- **Anthropic:** Called during Field Notes upload. 
  - *Retry Strategy:* 3 retries with exponential backoff. 
  - *Failure:* If it fails or triggers safety filters, return graceful error "The AI is resting. Try another meme."
- **Vercel Blob:** Direct upload from client using short-lived tokens generated by server to minimize server bandwidth.

## Performance Targets
- LCP (Largest Contentful Paint) < 1.5s on landing page.
- TTFB (Time to First Byte) < 200ms for `/api/memes/feed`.
- **Zero-Latency Swiping:** Images must load instantly during swipe. Frame drops during swipe animation are unacceptable.

## Testing Strategy
- **Unit Tests:** Points calculation logic, SRS algorithm interval calculations.
- **E2E Tests:** The core Feed loop (swipe, guess, breakdown, points update).
- **Integration Tests:** Anthropic API prompt parsing (ensuring JSON structure is consistently returned).
