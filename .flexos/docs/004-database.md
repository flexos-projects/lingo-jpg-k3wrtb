# Database

## Stack Note
Using **Supabase (PostgreSQL)**. Relational data is essential for the points economy, unlocks ledger, and UGC attribution (royalties). Real-time sync is required for comments and instant points balance updates. 
**Security:** Row Level Security (RLS) is heavily utilized. Users can only read their own `attempts` and `saves`. The `pointsBalance` can only be mutated via secure server-side RPC calls (not direct client inserts) to prevent cheating.

## Position/Computed Fields
- `pointsBalance`: Denormalized on the `users` table for fast reads. Updated via a PostgreSQL Trigger on `INSERT` to `attempts` (add points) and `unlocks` (subtract points). This ensures atomic transactions and prevents double-spending.

## Tier Enforcement
- `isPremium` emojis in The Vault can only be unlocked if `users.tier == 'pro'`. Enforced via Supabase RLS policies on the `unlocks` table.
- Field Notes uploads count checked against a daily limit for free users via server-side API before writing to `memes`.

## File Storage
- **Vercel Blob** for meme images and emoji assets.
- Images: max 5MB, auto-compressed to WebP on upload.
